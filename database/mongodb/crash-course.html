# $set operator

you just want replace you know one or two fields

db.posts.update(
  { title: 'Post Two' },
  {
    $set: {
      body: 'Body of post 2',
      category: 'Technology',
    },
  }
)
// WriteResult({ "nMatched" : 1, "nUpserted" : 0, "nModified" : 1 })

# $inc operator

db.posts.update(
  { title: 'Post One' },
  {
    // origin 4, now 6
    $inc: { likes: 2 },
  }
)

# $rename operator

db.posts.update(
  { title: 'Post One' },
  {
    // rename likes field to 'views'
    $rename: { likes: 'views' },
  }
)

# remove

db.posts.remove({ title: 'Post Four' })
// WriteResult({ "nRemoved" : 1 })

# sub document

In a relational database, if we hava post usually we want some comments.

What we do is hava a separate table for comments, we hava to create a foreign key that links to a primay key on the other table.

In mongodb, we can simply embed the comments right inside of the post collection.

db.posts.update(
  { title: 'Post One' },
  {
    $set: {
      comments: [
        {
          user: 'Mary Williams',
          body: 'Comment One',
          date: Date(),
        },
        {
          user: 'Harry White',
          body: 'Comment Two',
          date: Date(),
        },
      ],
    },
  }
)
// WriteResult({ "nMatched" : 1, "nUpserted" : 0, "nModified" : 1 })

# $elemMatch

Find all Mary Williams's comments.

db.posts.find({
  comments: {
    $elemMatch: { user: 'Mary Williams' },
  },
})

# fuzzy search

$text: performs a text search on the content of the fields indexed with a text index

$search: A string of terms that MongoDB parses and uses to query the text index.

db.posts.find({
  $text: {
    $search: '"Post O"',
  },
})

# compare operator

# greater than

db.posts.find({

})

# greater than or equal

db.posts.find({

})

# smaller than

db.posts.find({
  views: { $lt: 3 },
})

# smaller than or equal

db.posts.find({
  views: { $lte: 3 },
})
