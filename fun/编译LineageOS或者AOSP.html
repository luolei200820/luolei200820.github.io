# 编译 LineageOS/AOSP

记录一次使用 Arch Linux + Docker 编译 Lineage

  • 为什么使用 Arch Linux？
    • 因为它轻量，可以节省一部分内存
  • 为什么使用 Docker？
    • AOSP 推荐使用 Ubuntu 14.04 环境进行编译，版本太老不适合直接装在电脑上。Docker 可以很方便地构建带有编译环境地镜像，同时它相比虚拟机占用资源极少。

# Arch Linux

安装略过。。

需要注意的是，编译过程中需要使用大量的内存资源，AOSP 推荐为 16GB 以上，如果不够可以考虑使用更多的 swap。源码及编译输出需要 200GB 以上的存储空间。

我 16GB 内存，开了 16 个并发，结果 swap 占用到了将近 16GB，速度也是巨慢无比，花费了 6 个小时 20 分钟才编译完

内存不够了！！

# Docker

# 安装

$ sudo pacman -S docker
// 添加普通用户到docker组，使docker可以以普通用户运行
$ sudo usermod -aG docker <username>

# 修改配置文件

修改 docker 配置文件，以使用国内代理拉取镜像，修改 docker 镜像目录（可选）

/etc/docker/daemon.json

{
  "data-root": "/var/lib/docker",
  "registry-mirrors": [
    "https://registry.cn-hangzhou.aliyuncs.com",
    "https://mirror.ccs.tencentyun.com",
    "http://hub-mirror.c.163.com"
  ]
}

# 构建镜像

参考 AOSP 的Dockerfile进行一些修改

  1. 新建目录build-docker

  2. 将 git 配置文件拷贝到此处，重命名为gitconfig

  3. 新建sources.list，替换为阿里云镜像

deb https://mirrors.aliyun.com/ubuntu/ trusty main restricted universe multiverse
deb-src https://mirrors.aliyun.com/ubuntu/ trusty main restricted universe multiverse
deb https://mirrors.aliyun.com/ubuntu/ trusty-security main restricted universe multiverse
deb-src https://mirrors.aliyun.com/ubuntu/ trusty-security main restricted universe multiverse

deb https://mirrors.aliyun.com/ubuntu/ trusty-updates main restricted universe multiverse
deb-src https://mirrors.aliyun.com/ubuntu/ trusty-updates main restricted universe multiverse

deb https://mirrors.aliyun.com/ubuntu/ trusty-backports main restricted universe multiverse
deb-src https://mirrors.aliyun.com/ubuntu/ trusty-backports main restricted universe multiverse

## Not recommended
# deb https://mirrors.aliyun.com/ubuntu/ trusty-proposed main restricted universe multiverse
# deb-src https://mirrors.aliyun.com/ubuntu/ trusty-proposed main restricted universe multiverse

  1. 使用代理下载预编译的JDK8，重命名为jdk8-master.tar.gz

  2. 编写Dockerfile

FROM ubuntu:14.04
ARG userid
ARG groupid
ARG username

COPY sources.list /etc/apt/sources.list
RUN apt-get update && apt-get install -y git-core gnupg flex bison gperf build-essential zip curl zlib1g-dev gcc-multilib g++-multilib libc6-dev-i386 lib32ncurses5-dev x11proto-core-dev libx11-dev lib32z-dev ccache libgl1-mesa-dev libxml2-utils xsltproc unzip python openjdk-7-jdk
RUN apt-get install -y bc bison build-essential ccache curl flex g++-multilib gcc-multilib git gnupg gperf imagemagick lib32ncurses5-dev lib32readline-dev lib32z1-dev liblz4-tool libncurses5 libncurses5-dev libsdl1.2-dev libssl-dev libxml2 libxml2-utils lzop pngcrush rsync schedtool squashfs-tools xsltproc zip zlib1g-dev libwxgtk2.8-dev

COPY jdk8-master.tar.gz jdk8.tgz
RUN tar -zxf jdk8.tgz linux-x86 \
 && mv linux-x86 /usr/lib/jvm/java-8-openjdk-amd64 \
 && rm -rf jdk8.tgz

RUN curl -o /usr/local/bin/repo https://mirrors.tuna.tsinghua.edu.cn/git/git-repo \
 && chmod a+x /usr/local/bin/repo

RUN groupadd -g $groupid $username \
 && useradd -m -u $userid -g $groupid $username \
 && echo $username >/root/username \
 && echo "export USER="$username >>/home/$username/.gitconfig
COPY gitconfig /home/$username/.gitconfig
RUN chown $userid:$groupid /home/$username/.gitconfig
ENV HOME=/home/$username
ENV USER=$username

ENTRYPOINT chroot --userspec=$(cat /root/username):$(cat /root/username) / /bin/bash -i
  1. 构建镜像

docker build --build-arg userid=$(id -u) --build-arg groupid=$(id -g) --build-arg username=$(id -un) -t android-build-trusty .

此命令会将你当前的用户 id 和组 id 作为容器启动时的用户。

  1. 启动容器

docker run -it --rm -v $ANDROID_BUILD_TOP:/src android-build-trusty

-v将你下载的安卓源码目录绑定到容器的/src目录，-it分配一个伪终端，--rm在容器结束运行时销毁。

# 开始编译

$ source build/envsetup.sh
$ lunch
// 选择你要编译的机型
$ mka bacon <-jN> // -jN指定并发工作数量，如果是LineageOS
$ m <-jN> // 如果是AOSP
