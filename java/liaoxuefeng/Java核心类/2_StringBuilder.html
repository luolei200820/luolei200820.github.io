# StringBuilder

Java可以使用+来拼接字符串，但是在循环中每次都会创建新的字符串对象，然后扔掉旧的字符串对象。这样绝大部分字符串都是临时拼接对象，不但浪费内存， 还会影响GC效率。

为了高效拼接字符串，提供了StringBuilder类

StringBuilder sb = new StringBuilder(1024);
for (int i = 0; i < 1000; i++) {
    sb.append(',');
    sb.append(i);
}
String s = sb.toString();

还可以链式操作

public class Main {
    public static void main(String[] args) {
        StringBuilder sb = new StringBuilder(1024);
        sb.append("Mr ")
          .append("Bob")
          .append("!")
          .insert(0, "Hello, ");
        System.out.println(sb.toString());
    }
}

可以链式调用是因为append()方法返回了this

注意对于普通字符串的+操作，不需要改写，Java编译器做了优化，运行时会把字符串连接操作优化为数组复制或者StringBuilder

早期有个StringBuffer，线程安全同步版本，效率较低。
