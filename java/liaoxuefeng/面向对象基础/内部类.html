# 内部类

# Inner Class

public class Test {
    public static void main(String[] args) {
        Outer outer = new Outer();
        Outer.Inner inner = outer.new Inner();
        inner.hello();
    }
}

class Outer {
    private String name = "OuterName";

    class Inner {
        private String name = "InnerName";

        void hello(){
            System.out.println(Outer.this.name);
            System.out.println(this.name);
        }
    }
}

从上面代码可以看出：

  1. Inner Class必须借助Outer Class的实例才能实例化，这是因为它除了有一个this指向自己的实例，还隐含地持有一个Outer Class实例，可以用Outer.this访问这个实例。
  2. Inner Class可以访问和修改Outer Class的private字段，因为Inner Class的作用域在Outer Class内部，所以能访问Outer Class的private字段和方法。

观察Java编译器编译后的.class文件，可以发现Outer类被编译成Outer.class，而Inner类被编译成了Outer$Inner.class。

# Anonymous Class

匿名类，不用class关键字来定义的类。

Thread t = new Thread() {
    @Override
    public void run() {
        System.out.println("test thread");
    }
};

观察上面代码，匿名类继承自Thread类，并重写了run方法。除了可以继承自普通类，匿名类也可以实现接口。

Runnable r = new Runnable() {
    @Override
    public void run() {
        System.out.println("run");
    }
};

Runable是接口，接口是不能实例化的，上面代码其实是定义了一个实现了Runable接口的匿名类，并且通过new实例化该匿名类，然后转型为Runable。

因此匿名类在定义的时候就必须要实例化它。

匿名类可以少写很多代码，因为我们并不关心具体的类名，而仅仅只是要重写基类的方法或者实现某个接口。

同样，匿名类和内部类一样可以访问外部类的private字段。

观察编译后的.class，匿名类被编译成Outer$1.class，如果有多个依次2、3。。

# Static Nested Class

最后一种是静态内部类，用修饰符static修饰的内部类。

是完全独立的类，不在依附Outer类的实例，不能访问实例变量，只能访问静态变量。
