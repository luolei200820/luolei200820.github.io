# 包作用域

位于同一个包的类，可以访问包作用域的字段和方法。不用public、protected、private修饰的字段和方法就是包作用域。

对于顶级类（外部类）来说，只有两种修饰符：public和默认。public在任何地方都能访问，默认则要求在同一个包中。

在代码中，当编译器遇到一个class名称时：

  • 如果时完整类名，就直接根据完整类名查找这个class
  • 如果是简单类名，则按下列顺序依次查找：
    • 查找当前package是否存在这个class
    • 查找import的包是否包含这个class
    • 查找java.lang包是否包含这个class

最佳实践

为了避免名字冲突，需要确定唯一的包名，推荐的做法是使用倒置的域名来确保唯一性。

# 方法重载

规则：

  • 方法名称必须相同
  • 参数列表必须不同（个数不同或种类不同、参数排列顺序不同等）
  • 方法的返回类型可以相同也可以不相同
  • 仅仅返回类型不同不足以成为方法的重载

实现理论：

  • 方法名称相同时，编译器会根据调用方法的参数个数、参数类型等去逐个匹配，以选择对应的方法，如果匹配失败，则编译器报错。

# 冒泡排序

public static void babelSort(int[] a) {
    boolean flag = false;

    for (int i = 0; i < a.length - 1; i++) {
        for (int j = 0; j < a.length - 1 - i; j++) {
            if (a[j] > a[j + 1]) {
                int temp = a[j + 1];
                a[j + 1] = a[j];
                a[j] = temp;
                flag = true;
            }
        }
        if(!flag) return; // 带优化
    }
}

静态字段和静态方法

类的静态字段存储在公共区，不推荐使用实例变量来访问静态字段，可以使用类名.静态字段来访问静态对象。

同理有静态字段也就有静态方法，用static修饰的方法叫做静态方法。静态方法属于类，而不属于实例，因此方法内部无法访问this变量，也无法访问实例字段，只能访问静态字段。

接口的静态字段

接口是一个纯抽象类，因此不能有实例字段，但是可以有静态字段，并且静态字段必须为final类型

# 抽象类

  • 在class前加abstract就成抽象类了
  • abstract加在方法名称前，就成抽象方法了，抽象方法必须由它的子类来实现。
  • 抽象类不能实例化，但是可以有构造方法，被继承类调用
  • 抽象方法不能在普通类中
  • 构造方法不能声明为抽象方法

# 接口

接口就是规范，定义的是一组规则。

public interface UserService {
    // 接口中的所有定义其实都是抽象的public
    void run();

    // 常量 public static final
    int AGE = 99;
}

作用：

  1. 约束
  2. 定义一些方法，让不同的人实现
  3. public abstract
  4. public static final
  5. 接口不能实例化，接口中没有构造方法
  6. implements可以实现多个接口
  7. 必须要重写接口中的方法

可以定义default方法，作用是在给接口新增加方法时不必重写每个子类，只需要在需要覆写方法的子类中覆写即可。（JDK>=1.8）

interface Person {
    String getName();
    default void run() {
        System.out.println(getName() + " run");
    }
}

# 成员内部类

public class Outer {
    private int id = 10;
    public void out() {
        System.out.println("这是外部类的方法");
    }
    public class Inner{
        public void in(){
            System.out.println("这是内部类的方法");
        }
        //可以获得外部类的私有属性
        public void getId(){
            System.out.println(id);
        }
    }
}

使用内部类

Outer outer = new Outer();
Outer.Inner inner = new outer.new Inner();
inner.getId();

要实例化一个内部类，必须先创建一个Outer类的实例，然后调用Outer类实例的new来创建Inner类实例。

这是因为Inner Class除了有一个this指向自己，还隐含的有一个Outer Class实例，可以用Outer.this访问。

相比于普通类，内部类可以访问Outer类的private字段。

常见的用法：

public class Main {
    public static void main(String[] args) {
        Outer outer = new Outer("Nested");
        outer.asyncHello();
    }
}

class Outer {
    private String name;

    Outer(String name) {
        this.name = name;
    }

    void asyncHello() {
        Runnable r = new Runnable() {
            @Override
            public void run() {
                System.out.println("Hello, " + Outer.this.name);
            }
        };
        new Thread(r).start();
    }
}

观察asyncHello()方法，在方法内部实例化了一个Runnable。Runnable本身是接口，接口是不能实例化的，所以这里实际上是定义了一个实现了Runnable接口的匿名类，并通过new来实例化，然后转型为Runnable。

使用匿名类是因为不关心类名，可以少很多代码，和内部类一样，也可以访问private字段。

匿名类也可以继承自普通类，如HashMap

# 异常

分类

  • 检查性异常：最具代表性的是用户错误引起的，如打开一个不存在的文件，在编译时不能被简单忽略。
  • 运行时异常：可以被程序员避免，运行时异常可以在编译时被忽略。
  • 错误ERROR：错误不是异常，而是脱离程序员控制的问题，错误在代码中常被忽略。例如当栈溢出的时候，一个错误就发生了，编译时也检查不到。
public static void main(String[] args) {
    int a = 1;
    int b = 0;

    try {
        System.out.println(a / b);
    } catch (ArithmeticException e) {
        // catch 捕获异常
        System.out.println("程序出现异常，变量b不能为0");
    } finally {
        System.out.println("finally");
    }
}

可以写多个catch，从小到大地去捕获。

Ctrl + Alt + T 快捷键可以自动生成捕获异常。

使用throw关键字主动抛出异常，一般在方法中使用。

public void test(int a,int b) throws ArithmeticException{
    if(b==0){
        throw new ArithmeticException();// 主动抛出异常，一般在方法中使用。
    }
}

# 自定义异常

public class MyException extends Exception {
    // 传递数字>10
    private int detail;

    public MyException(int a) {
        this.detail = a;
    }

    // toString：异常地打印信息
    @Override
    public String toString() {
        return "MyException{" + "detail=" + detail + '}';
    }
}

public static void main(String[] args) {
    try {
        test(100);

    }
}

# classpath和jar

不需要设置classpath，那样做只会污染环境变量，正确的做法是在启动JVM的时候通过命令行参数传入。

在IDE中运行Java程序，IDE自动传入的-cp参数是当前工程的bin目录和引入的jar包。

java -cp .;C:\work\project1\bin;C:\shared abc.xyz.Hello

DANGER

不要把任何Java核心库添加到classpath中！JVM根本不依赖classpath加载核心库！

jar包

编译出的.class文件散落在各层目录中，不便于管理，将目录压缩成.zip然后再把它改成.jar就做成了一个jar包。

注意目录结构应该和包名一致。

要执行jar包中的某个类：

java -cp ./hello.jar abc.xyz.Hello

这样JVM会自动再hello.jar文件中去搜索某个类

特殊的/META-INF/MANIFEST.MF文件：

MANIFEST.MF是纯文本，可以指定Main-Class和其它信息。JVM会自动读取这个文件，如果存在Main-Class就不必在启动时指定类名。

java -jar hello.jar

jar包还可以包含其它jar包，这个时候，就需要在MANIFEST.MF文件里配置classpath了。

在大型项目中，不可能手动编写MANIFEST.MF文件，再手动创建zip包。Java社区提供了大量的开源构建工具，例如Maven，可以非常方便地创建jar包。
