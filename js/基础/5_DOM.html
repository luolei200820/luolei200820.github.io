# DOM

# 节点与元素

在 HTML DOM 中，每一个元素都是一个节点：

  • 文档是一个文档节点
  • 所有的 HTML 元素都是元素节点
  • 所有 HTML 属性都是属性节点
  • 文本插入到 HTML 元素是文本节点
  • 注释是注释节点

元素对象的子节点可以是元素节点、文本节点、注释节点

# DOM Document 对象

实例对象：document

# 属性

  • documentElement —— 返回 html 元素
  • body —— 返回 body 元素
  • ...

# 方法

  • createElement()
  • getElementById()
  • getElementByClassName()
  • getElementByTagName()
  • querySelector() *推荐，语法类似 CSS 选择器
  • ...

# DOM Element 对象

实例对象：所有的 HTML 标签

# 属性

  • nodeName 返回元素的标记名（大写）
  • nodeType 返回元素的节点类型（是一个数字）
  • nodeValue 返回元素的节点值
  • nextSibling 该元素紧跟的一个节点
  • firstChild 返回元素的第一个子节点
  • childNodes 返回元素的一个子节点的数组
  • parentNode 返回元素的父节点

# 元素的增删改

  • appendChild()
  • insertBefore()
  • removeChild()
  • replaceChild()

# 获取/设置样式

  • style.<属性名>

  • currentStyle.<属性名> 只有 IE 支持（不能修改）

  • windows.getComputedStyle() （不能修改）

    • 参数为元素的引用，返回一个对象，对象中封装了当前元素对应的样式

  • clientWidth

  • offsetWidth

  • offsetParent 返回一个指向最近的（指包含层级上的最近）包含该元素的定位元素

  • offsetLeft

    返回当前元素左上角相对于定位父元素的左边界偏移量。然而对于截断到下一行的行内元素（如 span）

    <div
  style="width: 300px; border-color:blue;border-style:solid;border-width:1;"
>
  <span>Short span. </span>
  <span id="long">Long span that wraps withing this div.</span>
</div>

<div
  id="box"
  style="position: absolute; border-color: red;
  border-width: 1; border-style: solid; z-index: 10"
></div>

<script>
  var box = document.getElementById('box')
  var long = document.getElementById('long')
  //
  // long.offsetLeft这个值就是span的offsetLeft.
  // long.offsetParent 返回的是body（在chrome浏览器中测试）
  // 如果id为long的span元素的父元素div，设置了position属性值，只要不为static,那么long.offsetParent就是div

  box.style.left = long.offsetLeft + document.body.scrollLeft + 'px'
  box.style.top = long.offsetTop + document.body.scrollTop + 'px'
  box.style.width = long.offsetWidth + 'px'
  box.style.height = long.offsetHeight + 'px'
</script>

  • scrollWidth 整个水平滚动区域的宽度

  • scrollHeight 整个垂直滚动区域的高度

  • scrollLeft 获取水平滚动的距离

  • scrollTop 获取垂直滚动的距离

Chrome 认为浏览器的滚动条是 body 的，可以通过 body.scrollTop 来获取 火狐认为浏览器的滚动条是 html 的

# Events 对象

响应函数

在事件的响应函数中，响应函数是给谁绑定的，this 就是谁

有两种添加方式：

  1. 属性方式

    btn.onclick = function () {
  alert('hello world')
}

  2. addEvenetListener 方式

    btn.addEventListener('click', function () {
  alert('hello world')
})

    addEventListener 有三个参数

    • 事件的字符串
    • 回调函数
    • 是否在捕获阶段触发事件，需要一个布尔值，一般传 false

  3. 区别：

    属性的方式只能添加一个响应函数，addEvenetListener 方式可以添加多个,当事件触发时按绑定的顺序执行

# 事件对象

当事件的响应函数被触发时，浏览器每次都会将一个事件对象作为实参传递进响应函数 在事件对象中封装了当前事件的一切信息，比如鼠标的坐标，键盘哪个按键被按下

# 鼠标事件

onmousemove事件

  • clientX 鼠标指针在当前可见窗口的坐标
  • clientY
  • pageX
  • pageY 鼠标指针相对于页面的坐标

# 键盘事件

一般绑定到可以获取到焦点的对象或 document onkeydown

  • 如果一直按住按键，则事件会连续触发
  • 连续触发时，第一次和第二次之间的间隔稍慢一些，之后很快 onkeyup

# 键盘事件属性

  • keyCode —— 键盘按键对应的编码
  • altKey
  • ctrlKey
  • shiftKey
  • 当需要同时按下功能键和其他按键时，使用上面三个属性

如判断 y 和 ctrl 同时按下

event.keycode === 89 && event.ctrlKey

# 事件冒泡

所谓的事件冒泡是指事件的向上传导，当后代元素上的事件被触发时，其祖先元素的相同事件也会被触发

取消冒泡： 可以将事件对象的 cancleBubble 设置为 true event.cancleBubble = true;

# Event delegation（事件委托）

Event delegation allows you to avoid adding event listeners to specific nodes; instead, the event listener is added to one parent.
事件委托允许避免你将事件监听器添加到特定的节点，相反，事件监听器添加到父节点。

That event listener analyzes bubbled events to find a match on child elements.
事件监听器分析冒泡的事件来找到一个符合的孩子元素。

<ul id="parent-list">
  <li id="post-1">Item 1</li>
  <li id="post-2">Item 2</li>
  <li id="post-3">Item 3</li>
  <li id="post-4">Item 4</li>
  <li id="post-5">Item 5</li>
  <li id="post-6">Item 6</li>
</ul>

when the event bubbles up to the ul element, you check the event object's target property to gain a reference to the actual clicked node.
当事件冒泡到ul元素时，你检查 event 对象的 target 属性来获取真实点击节点的引用。

// Get the parent DIV, add click listener...
document.getElementById('myDiv').addEventListener('click', function (e) {
  // e.target was the clicked element
  if (e.target && e.target.matches('a.classA')) {
    console.log('Anchor element clicked!')
  }
})

# 事件的传播

W3C 将事件传播分成了三个阶段

  1. 捕获阶段

    在捕获阶段时从最外层的祖先元素，向目标元素进行事件的捕获，但是默认此时不会触发事件，如果希望在捕获阶段就执行事件，可以将 addEventListener 的第三个参数设置为 true。一般情况下不会设置。

  2. 目标阶段

    事件捕获到目标元素，捕获结束开始在目标元素上触发事件

  3. 冒泡阶段

    事件从目标元素向他的祖先元素传递，依次触发祖先元素上的事件
