# 继承与原型链

JavaScript 的继承是基于原型的
每一个实例对象（object）都有一个私有属性，称之为__proto__，指向它的构造函数的原型对象（prototype）
原型对象也有私有属性__proto__，也指向它的构造函数的原型对象，层层向上直到一个对象的原型对象为null
根据定义，null没有自己的原型，因此是原型链中的尽头

TIP

someObject.[[Prototype]]是 ECMAScript 标准，__proto__是非标准，但是大多数浏览器实现

# 显式原型属性与隐式原型属性

  1. 显式原型属性
    prototype是每个函数对象的属性，其默认值为空的 Object 实例对象，称为原型对象。
  2. 隐式原型属性
    __proto__是每个实例对象的属性，当使用构造函数生成实例对象时，会自动将Fn.prototype赋值给fn.__proto__
    隐式原型不能修改，只能修改显式原型，一般添加实例对象共享的方法

# 一些特殊的原型链

  1. 所有函数对象的__proto__属性都指向 Function.prototype。

    Function.__proto__ === Function.prototype // true
Object.__proto__ === Function.prototype // true

  2. Object 原型对象不是 Object 的实例对象

    Object.prototype instanceof Object // false

    原因见探索 instanceof

# 属性问题

  1. 读取对象的属性值时，会自动到原型链中查找
  2. 设置对象属性值时，不会查找原型链，如果对象本身上没有此属性，则直接添加
  3. 方法一般定义在原型中，属性一般定义在对象本身上

# 探索 instanceof

instanceof 左侧是实例对象，右侧是函数对象，实例对象则沿着原型链向上探索，如果原型链中含有函数对象的原型，则表明实例对象是函数对象的实例。

# prototype 重新赋值对象

如果函数对象的 prototype 被重新赋值为一个新的对象
则在修改前，使用此构造函数创建的实例对象，__proto__属性没有指向新的对象，无法访问新对象的属性。
而在修改后，使用此构造函数创建的实例对象，__proto__属性指向了新的对象，可以访问新对象的属性。
