# 变量提升和函数提升

一个简单的变量提升例子

var a = 3
function fun() {
  console.log(a)
  var a = 4
}
fun() // 输出undefined

fun2() // 输出fun2
function fun2() {
  console.log('fun2')
}

# 变量提升和函数提升的优先级

当遇到函数和变量同名且都被提升的情况，函数声明的优先级比较高，因此变量声明会被函数声明所覆盖，但是可以重新被赋值

function a() {}
var a
console.log(typeof a) // 'function'

if (b in window) {
  var b = 1
}
console.log(b) // 1

// 相当于
var b
if (b in window) {
  b = 1
}
console.log(b)

var c = 1
var test = function c(c) {
  // 表达式声明方式
  console.log(c)
}
c(3) // 报错，因为c被赋值1，不再是函数了
test() // undefined

# 执行上下文

全局执行上下文

只有一个，在浏览器中就是windows对象

函数执行上下文

JS 代码首次运行，都会首先创建一个全局执行上下文压入执行栈中，之后每当有函数调用，都会创建一个新的函数执行上下文并压入栈中。执行栈 LIFO，可以理解为当 JS 代码执行完之前栈底都有一个全局执行上下文。

eval 执行上下文

并不常用，暂不讨论

# 执行上下文栈

调用函数时，会在内存栈的栈顶创建函数执行上下文对象
若函数没有执行完则函数执行上下文对象不会消失
栈底永远是全局执行上下文 栈顶指向的是正在运行的函数

# 作用域与作用域链

与执行上下文的区别

  1. 全局作用域之外，每个函数都会创建自己的作用域，作用域在函数定义的时候创建，而执行上下文在函数调用之前。
  2. 作用域是静态的，只要函数定义了就一直存在，不会改变。执行上下文是动态的，在调用函数时创建，调用结束时释放。

# 执行上下文的生命周期

包括三个阶段：创建->执行->回收

  1. 创建阶段

    当函数被调用，但还未执行内部代码时，会做以下三件事

    • 创建变量对象，形参由实参赋值，初始化 arguments，提升函数声明和变量声明
    • 创建作用域链，用以解析变量，当被要求解析变量时，从代码嵌套的最内层开始，查找变量，如果未找到就会跳到上一层父级作用域中查找，直到查找到该变量。
    • 确定 this 的指向，参见下节

    在一段 JS 代码执行之前，要先解析代码，解析时会创建一个全局上下文环境，事先把变量声明、函数声明拿出来，变量暂时赋值为 undefined，函数声明后即可使用。之后再执行程序。

    函数执行之前也会创建函数执行上下文，跟全局执行上下文差不多，不过函数执行上下文中会多出现函数的参数、arguments

  2. 执行阶段

    执行变量的赋值、代码的执行

  3. 回收阶段

    执行上下文出栈等待垃圾回收器回收

# 确定 this 的指向

一个重要概念：this 的指向是在执行时确定的，而不是在定义时

  • 对于直接调用的函数，this 时 window
  • 对于对象.函数名()来说，谁调用的 this 就是谁
  • 在构造函数中，this 指的是实例对象
  • call、apply 和 bind，this 是第一个参数
  • 箭头函数没有自己的 this，向外查找执行上下文中的 this
