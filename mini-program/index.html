# 小程序登录

小程序可以通过微信官方提供的登录能力方便地获取微信提供的用户身份标识，快速建立小程序内的用户体系。

# 登录流程时序

DETAILS

openid：用户在当前小程序中的唯一标识
unionid：用户在微信开放平台的唯一标识（如需要在拥有的多个公众号、小程序中区分同一个用户身份）
session_key：用于服务端解密开放数据

# 自定义登录态

在以往的 web 应用中，验证用户的身份一般流程是这样的

  1. 用户向服务器发送用户名和密码
  2. 服务器验证正确之后，在当前会话（session）中保存相关的信息，如用户的角色、登录时间等。
  3. 服务端将 session_id 写入用户的 Cookie 中
  4. 以后用户请求都会带上 Cookie，从而服务端知道用户身份

这么做的问题在于拓展性不好，因为 Cookie 有跨域问题，这可以用其他发送方式解决。另一种解决方式是将 session_id 持久化，写入到数据库中，每个网站收到请求后都访问持久层，容易造成单点故障。

# 使用 JWT 来用户认证

将标识用户唯一的身份信息写入 JWT，发送给用户，用户每次请求接口、资源时都带上，服务端通过签名认证这个数据是自己的，同时也没有被篡改。

WARNING

  1. JWT 默认是不加密的，不能将用户敏感的信息存入，但也可以加密后使用
  2. 为了减少盗用 JWT 的有效时间应该设置得比较短，对于重要的权限要重新认证
  3. 避免使用 HTTP 明文传输

# JWT 续期机制

在登录时签发 accessToken 和 refreshToken，都传给用户。accessToken 过期时间一般设置 10-15 分钟或者更长；refreshToken 不设置过期时间，只能手动使其失效，或者也可以设置较长的时间，当失效后需要重新登录，因此需要把 refreshToken 存入持久层。

当请求接口发现 accessToken 过期后，使用 refreshToken 重新获取 accessToken。当 refreshToken 过期时重新登录。

# 小程序调用 wx.login()

如要静默登录，在小程序页面 onload 时使用 wx.login()接口获取用户登录凭证，将其发送给服务端

  1. 检查缓存中是否有 accessToken
Page({
  silentLogin() {
    if (!wx.getStorageSync('accessToken')) {
      wx.login({
        success(res) {
          if (res.code) {
            // 将code发送给开发者服务器
          } else {
            console.log('登录失败', res.errMsg)
          }
        },
      })
    }
  },
  onLoad() {
    this.silentLogin()
  },
})
  1. 发送 code
wx.request({
  url: 'http://localhost:3000/api/user/login',
  method: 'POST',
  // 把临时登录凭证发往开发者服务器
  data: {
    code: res.code,
  },
  success(response) {
    // response中含有accessToken和refreshToken
    // 需要保存到缓存中
  },
})

# 服务端调用 auth.code2Session

这一步是调用微信服务器的 auth.code2Session 接口，用用户的登录凭证换取 openid、unionid、session_key。

  1. 使用 axios 发送请求
router.post('/login', (req, res, next) => {
  if (!req.body.code) return res.sendStatus(403)

  // 调用微信服务器的登陆凭证验证接口
  axios
    .get('https://api.weixin.qq.com/sns/jscode2session', {
      params: {
        // 参数为AppID(小程序ID)和AppSecret(小程序密钥)
        // 可以在微信公众平台查看
        appid: process.env.APPID,
        secret: process.env.APPSECRET,
        js_code: req.body.code, // 用户登录临时凭证code
        grant_type: 'authorization_code', // 文档固定写法
      },
    })
    .then((response) => {
      // 响应中包含openid、session_key
    })
    .catch((err) => {
      next(err)
    })
})
  1. 将响应中的 openid 与数据库中的对比
// 检查数据库中是否已经存在此用户
let user = users.find((u) => u.openid === response.data.openid)
if (!user) {
  // 不存在此用户，保存下来
  user = {
    session_key: response.data.session_key,
    openid: response.data.openid,
  }
  users.push(user)
}
  1. 生成 accessToken 和 refreshToken
let payload = {
  openid: user.openid,
}
// 生成accessToken和refreshToken
// 自定义登录态与openid,sesson_key关联
const accessToken = generateAccessToken(payload)
const refreshToken = generateRefreshToken(payload)
res.json({
  accessToken,
  refreshToken,
})

# 小程序将 token 缓存

success(response) {
  // 设置本地缓存
  console.log('成功获取token')
  // 1. 以后请求接口，需要将accessToken放入http请求头authorization字段
  // 2. 若发现accessToken失效了，则要用refreshToken调用/api/user/token重新获取
  wx.setStorageSync('accessToken', response.data.accessToken)
  wx.setStorageSync('refreshToken', response.data.refreshToken)
}

# 请求头中带上 token

wx.request({
  url: 'http://localhost:3000/posts',
  header: { Authorization: `Bearer ${wx.getStorageSync('accessToken')}` },
  success(res) {
    //...
  },
})

# token 续期

router.post('/token', (req, res) => {
  const refreshToken = req.body.token
  // refreshTokens存在数据库里，或redis
  if (!refreshTokens.includes(refreshToken)) return res.sendStatus(403)

  jwt.verify(refreshToken, process.env.REFRESH_TOKEN_SECRET, (err, user) => {
    if (err) return res.sendStatus(403)
    const accessToken = generateAccessToken({ openid: user.openid })
    res.json({ accessToken })
  })
})

# 撤销 token

router.delete('/logout', (req, res) => {
  refreshTokens = refreshTokens.filter((token) => token !== req.body.token)
  res.sendStatus(204)
})
